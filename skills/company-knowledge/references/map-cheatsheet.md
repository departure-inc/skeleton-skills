# Domain cheatsheet

`$ROOT/_general/` 配下。まず領域の README を読んでから個別ファイルへ。

| 領域 | 典型トリガー | Path |
|------|----------------|------|
| human | 振る舞い、コミュニケーション、議事 | `human/` |
| method | 営業、見積、PM、スクラム、設計手法 | `method/` |
| tech | 言語、FW、クラウド、コーディング標準 | `tech/` |
| team | チーム開発の進め方 | `team/` |
| tool | 業務ツール紹介 | `tool/` |
| uml | UML の書き方 | `uml/` |
| docs | 設計書サンプル・標準化 | `docs/` |
| architecture | よくある機能構成の参考 | `architecture/` |
| agent | AI エージェント関連の社内メモ | `agent/` |
| ospo | OSS / OSPO | `ospo/` |

スタック別のテンプレや実装骨格は `_general` 外（例: `$ROOT/rails/`、`$ROOT/nextjs/`）にもある。地図を読んでから必要なら辿る。
